Shortcodes

Wherever you want to render a widget, you can use the [widgetkit ...] shortcode that will be replaced by the actual widget in the final browser output. To render a widget, you always have to combine a content item with a widget type that defines the way the content is rendered.

A shortcode has to contain at least the two parameters id and widget

[widgetkit id="42" widget="slideshow"]
Parameter Value
id The unique id of the content you want to render.
widget The widget type to render: grid, grid-stack, map, slideshow and switcher - or other types added by your theme or yourself.

There is a number of other properties you can use in shortcodes. These depend on the actual widget type you have set as the value of the widget parameter. The easiest way to set these parameters is to use the Widgetkit available next to your content window. You can visually configure your widget. When confirming the configuration, the shortcode will be updated automatically to something more complex, i.e.:

[widgetkit id="23" widget="grid-stack" divider=1 media_border="circle" animation_media="fade" animation_content="slide"]

For further details have a look at the documentation and examples of the different {doc:widget-types text:widget types}
